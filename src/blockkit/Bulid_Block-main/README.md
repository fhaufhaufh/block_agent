# Airbot 智能视觉抓取系统 (Airbot Intelligent Visual Grasping System)

本项目实现了一个基于 **Airbot 机械臂**、**Intel RealSense 相机** 和 **YOLOv8** 的自动化抓取与放置（Pick-and-Place）系统。系统专为积木堆叠任务设计，采用了 **混合视觉算法 (Hybrid Vision)**，结合了深度学习的通用检测能力与传统计算机视觉的高精度姿态估计能力。

---

## 📂 1. 项目文件结构与作用 (Project Structure)

| **`config/camera.yaml`** | **系统配置文件**。存储 YOLO 模型路径、置信度阈值、相机分辨率设置以及至关重要的 **手眼标定外参矩阵 (`camera_extrinsics`)**。 |

| **`best.pt`** | **YOLOv8 模型权重**。经过训练的神经网络模型文件，用于识别积木类别和位置。 |

## 🧠 2. 核心算法原理 (Core Algorithms)

本系统采用多层级的感知策略来确保抓取的成功率和精度。

### 2.1 混合视觉感知 (Hybrid Vision Perception)
系统针对不同类型的物体采用不同的姿态估计策略：

1.  **粗定位 (YOLOv8-OBB)**:
    *   **输入**: RGB 图像。
    *   **处理**: 使用 YOLOv8 旋转目标检测 (OBB) 模型。
    *   **输出**: 物体类别 (Class ID)、置信度、中心坐标 $(u, v)$ 以及初步的旋转角度 $\theta_{yolo}$。
    *   **作用**: 快速在视野中找到目标，区分不同形状的积木。

2.  **精细定向 (PCA + HSV)**:
    *   **触发条件**: 针对特定需要高精度对齐的物体（如代码中的 `Class ID 2`）。
    *   **ROI 提取**: 根据 YOLO 的检测框裁剪出感兴趣区域 (ROI)。
    *   **HSV 分割**: 在 ROI 内利用 HSV 颜色空间阈值分割出物体掩膜 (Mask)，去除背景干扰。
    *   **PCA (主成分分析)**: 计算掩膜点云的协方差矩阵，分解出特征向量。第一主成分即为物体的精确主轴方向。
    *   **方向校正**: 通过分析轮廓点在主轴两侧的分布，消除 PCA 固有的 180° 方向模糊，确保抓取方向唯一。

### 2.2 视觉伺服对齐 (Visual Servoing Alignment)
为了消除透视畸变（Parallax Error）对坐标转换的影响，系统在抓取前执行 **闭环位置控制**：
1.  计算目标中心在图像中的像素偏差 $(\Delta u, \Delta v)$。
2.  将像素偏差转换为相机坐标系的位移向量。
3.  控制机械臂平移，使相机光轴垂直对准物体中心。
4.  **结果**: 此时物体位于图像中心，深度测量最准，且坐标转换误差最小。

### 2.3 鲁棒数据滤波 (Robust Filtering)
*   **深度滤波**: 融合多帧深度图，优先取中心点深度；若中心无效（值为0），则取邻域中值。
*   **位置滤波 (IQR)**: 在精细检测阶段，连续采集 10 次位姿数据。使用 **四分位距 (Interquartile Range, IQR)** 算法剔除离群值，取剩余数据的中位数作为最终抓取目标。
*   **角度平滑**: 使用 `AngleSmoother` 类（基于三角函数的滑动平均）平滑连续帧的角度波动。

## 💻 3. 代码整体流程 (Main Code Flow)

`main.py` 的执行流程设计为状态机模式，主要包含以下步骤：

### 3.1 系统初始化 (`__init__`)
1.  **加载配置**: 读取 `config/camera.yaml`。
2.  **连接设备**:
    *   初始化 `AIRBOTPlay` 机械臂连接 (Port 50000)。
    *   启动 `RealSense` 相机流，开启高精度模式 (High Accuracy)，并设置 RGB-Depth 对齐。
3.  **加载模型**: 载入 YOLOv8 `best.pt` 权重。

### 3.2 检测循环 (`run_detection`)
该函数是感知的核心入口，包含两个阶段：

*   **阶段一：闭环对齐 (Alignment Phase)**
    *   **循环**: 最多尝试 5 次。
    *   **检测**: 采集图像 -> YOLO 推理 -> 获取最近目标。
    *   **判断**: 如果目标距离图像中心小于阈值 (15像素)，则认为对齐完成。
    *   **动作**: 计算偏差向量，控制机械臂平移修正位置。

*   **阶段二：精细估计 (Estimation Phase)**
    *   **采样**: 机械臂静止，连续采集 10 帧数据。
    *   **计算**:
        *   对每一帧执行 `process_frame_test`。
        *   如果是特定积木，启用 PCA 算法计算精确角度。
        *   计算相机坐标 -> 基座坐标的变换。
    *   **滤波**: 对 10 组 $(x, y, z, \theta)$ 数据应用 IQR 滤波和中值提取。
    *   **输出**: 得到最终的 **抓取位姿 (Grasp Pose)**。

### 3.3 抓取序列执行 (`execute_place_sequence`)
一旦获得抓取位姿，系统执行标准的 Pick-and-Place 动作：
1.  **悬停 (Hover)**: 移动到物体正上方 (Z + 15cm)。
2.  **下探 (Descend)**: 垂直下降到抓取高度 (Z + 3cm)。
3.  **抓取 (Grasp)**: 闭合夹爪 (`move_eef_pos([0.0])`)。
4.  **抬起 (Lift)**: 抬升至安全高度。
5.  **放置 (Place)**: 移动到预设的放置点，松开夹爪，并抬起。

## 🚀 4. 快速开始 (Quick Start)

### 环境要求
*   Windows / Linux
*   Python 3.8+
*   依赖库: `airbot_py`, `pyrealsense2`, `ultralytics`, `opencv-python`, `numpy`, `pyyaml`

### 运行步骤

1.  **启动机械臂**: 确保 Airbot 机械臂已上电并连接到电脑。
2.  **检查配置**: 确认 `config/camera.yaml` 中的 `camera_extrinsics` 与当前硬件安装一致。
3.  **运行主程序**:
    ```bash
    python main.py
    ```
4.  **操作说明**:
    *   程序启动后会自动连接机械臂和相机。
    *   机械臂将移动到初始观测位置。
    *   程序将按照预设的脚本顺序（抓取积木1 -> 2 -> 3 -> 4）自动执行任务。

---

## ⚠️ 注意事项

*   **光照影响**: PCA 和 HSV 算法对光照敏感。如果环境光照变化较大，请微调 `main.py` 中的 `self.block3_hsv_min` 和 `self.block3_hsv_max` 阈值。
*   **安全距离**: 请确保机械臂工作空间内无障碍物，以免发生碰撞。
*   **标定精度**: 抓取精度高度依赖于手眼标定的准确性。如果发现抓取位置有系统性偏差（如总是偏左 1cm），请重新运行 `calibration_hand_eye.py`。
